由于免费的SSL证书仅支持申请3个月的了，每过3个月都得重新手动申请并部署，有些许麻烦，故找到了一个自动化部署工具——certd。

安装

这里作者使用源码安装，需要安装node环境进行编译。

node

安装完毕后node -v可以看到安装的版本。

certd

从github下载最新的源码，下载完毕后，将源码解压，运行start.sh脚本（服务器内存低的请勿尝试）。

由于作者的服务器内存一共只有4g，部署服务后仅剩1.5g左右，运行start.sh会卡死在客户端npm run build的阶段（亲测），于是只能先在Windows中进行编译，再将编译后的文件上传到服务器，手动运行。

以下为start.sh的内容，可以按照其步骤依次执行，只是Windows下命令有点不同。

按照上述Windows的执行步骤后，将certd-server目录打包复制到服务器上，然后再解压，准备步骤就完成了。

运行

在上传的certd-server目录下运行npm run start，此时作者出现报错Error: Cannot find module 'cross-spawn'，如果你也遇到这个错误，手动删除node_modules和package-lock.json然后再运行npm install重新安装依赖即可。
